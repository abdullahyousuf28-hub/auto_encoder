# Data-Driven Fault Detection in Process Systems Using PCA and Autoencoders

This project presents a data-driven approach for fault detection in process systems using AutoEncoder-based deep learning and Principal Component Analysis (PCA). The objective is to evaluate and compare the effectiveness of nonlinear and linear monitoring techniques for detecting faults in complex industrial processes. The dataset used is obtained from an offshore gas processing plant simulation and consists of 105 multivariate process variables under normal and faulty operating conditions.The AutoEncoder is trained in an unsupervised manner using normal operation data to learn the intrinsic nonlinear relationships among process variables. Fault detection is performed using reconstruction residuals, where deviations from learned patterns indicate abnormal conditions. PCA is implemented as a benchmark method using Hotelling’s 𝑇2and Squared Prediction Error (SPE) statistics. Performance is evaluated using Fault Detection Rate (FDR), False Alarm Rate (FAR), Accuracy (ACC), and Time to Detect (TTD). The results show that the AutoEncoder achieves higher detection accuracy and faster response, particularly for nonlinear faults such as valve failures. While PCA performs adequately for faults with large variance changes, it shows limitations in capturing nonlinear process dynamics. Overall, the study demonstrates the effectiveness of nonlinear data-driven approaches for fault detection in complex systems.

# Model

The core of the methodology is the development of a deep AutoEncoder (AE) using PyTorch. The AutoEncoder is designed with an encoder–decoder architecture, where the encoder compresses the high-dimensional input data into a lower-dimensional latent space, and the decoder reconstructs the original data from this compressed representation. The model structure consists of an input layer (105 features), two hidden layers, and a latent layer, followed by a symmetric decoder. The architecture ensures dimensionality reduction, enabling the model to learn the underlying structure of normal process behavior. The model is trained using the mean squared error (MSE) loss function, which minimizes the difference between the input and reconstructed output. To ensure optimal model performance, hyperparameter tuning is conducted using Optuna. Key parameters, including hidden layer sizes, latent dimension, learning rate, batch size, weight decay, and activation functions, are systematically optimized. The AutoEncoder is trained using the Adam optimizer during each trial. A structural constraint (hidden1 > hidden2 > latent) is imposed to maintain progressive dimensionality reduction.
Each trial trains a new model and evaluates it using validation reconstruction error, while inefficient trials are pruned early to improve computational efficiency. This automated tuning ensures that the final model is not arbitrarily selected but is systematically optimized for best performance. After training, the AutoEncoder learns the normal operating behavior of the process.
